# Инструкция по сборке «Заменитель текста» в EXE

Полное руководство по сборке автономного исполняемого файла `ЗаменительТекста.exe`
для Windows с учётом всех зависимостей, включая внешние библиотеки для работы
с форматами AutoCAD (`.dwg` / `.dxf`).

## 1. Кратко: быстрая сборка

Если Python уже установлен, а внешние библиотеки на месте:

```cmd
cd make_exe
build.bat
```

Скрипт сам поставит зависимости, сгенерирует иконки и соберёт
`dist\ЗаменительТекста.exe`. Подробности — ниже.

---

## 2. Что собирается и из чего

| Компонент | Назначение | Как попадает в EXE |
|-----------|-----------|--------------------|
| `main.py`, `ui/`, `text_replacer.py` | код приложения | вшивается PyInstaller'ом |
| `python-docx`, `openpyxl`, `ezdxf` | Word / Excel / DXF | Python-пакеты |
| `pymupdf`, `pikepdf`, `pypdf` | PDF (тройной запас) | Python-пакеты + нативные DLL |
| `sv-ttk` | тема Sun Valley | Python-пакет (+ .tcl данные) |
| `Pillow` | иконки интерфейса | Python-пакет |
| `resources/` | PNG-иконки, `app.ico` | копируется в EXE через `.spec` |
| **`libredwg/`** | конвертер `.dwg` ↔ `.dxf` | копируется рядом/внутрь EXE |
| **ACadSharp** (опц.) | прямое чтение/запись `.dwg` | внешняя .NET-утилита |

Замена в `.dwg` устроена так: программа сначала пробует ACadSharp (если собран),
затем — LibreDWG (`dwg2dxf` → замена в DXF → `dxf2dwg`). Поэтому для полноценной
работы с `.dwg` нужна **хотя бы одна** из этих внешних библиотек. Word, Excel,
PDF и DXF работают без них.

## 3. Установка Python

1. Скачайте **Python 3.10 или новее** с [python.org](https://www.python.org/downloads/).
2. При установке **обязательно** включите галочку **«Add Python to PATH»**.
3. Проверьте:
   ```cmd
   python --version
   pip --version
   ```

---

## 4. Python-зависимости

Все Python-библиотеки перечислены в `requirements.txt`:

```cmd
python -m pip install --upgrade pip
python -m pip install -r requirements.txt
python -m pip install pyinstaller
```

Состав `requirements.txt`: `sv-ttk`, `python-docx`, `openpyxl`, `ezdxf`,
`pymupdf`, `pikepdf`, `pypdf`, `Pillow`.

## 5. Внешние библиотеки: LibreDWG и ACadSharp

Это самая важная часть для поддержки `.dwg`. Эти библиотеки **не ставятся через
pip** — их нужно подготовить отдельно. Исходники лежат в папке `prodact/`
(в репозитории Github они исключены — см. раздел 10).

### 5.1. LibreDWG (обязательный путь для `.dwg`)

LibreDWG предоставляет утилиты `dwg2dxf.exe` и `dxf2dwg.exe`. Программа ищет их
в папке `libredwg\` рядом с собой.

**Вариант А — взять готовые бинарники.**
Папка `libredwg/` уже содержит собранные `.exe` и зависимые `.dll`
(`libredwg-0.dll`, `libpcre2-*.dll`, `libiconv-2.dll`). Если они на месте —
ничего делать не нужно, `.spec` включит папку в сборку.

**Вариант Б — собрать самостоятельно** (если бинарников нет):
1. Установите [MSYS2](https://www.msys2.org/) и в его консоли:
   ```bash
   pacman -S mingw-w64-x86_64-toolchain mingw-w64-x86_64-libredwg
   ```
2. Скопируйте из `C:\msys64\mingw64\bin\`:
   `dwg2dxf.exe`, `dxf2dwg.exe`, `libredwg-0.dll`, `libpcre2-8-0.dll`,
   `libpcre2-16-0.dll`, `libiconv-2.dll` — в папку `libredwg\` проекта.

> **Проверка:** `libredwg\dwg2dxf.exe --version` должна вывести версию.

### 5.2. libdxfrw (запасной обработчик DXF)

В составе LibreDWG-пути приложение при сбое `ezdxf` пытается использовать
нативную обёртку `dxfrw_native` (исходник `prodact/libdxfrw/`). Это
необязательно: если обёртка не собрана, замена в DXF идёт через `ezdxf`.
Сборка обёртки описана в `prodact/libdxfrw/wrapper/BUILD.md`.

### 5.3. ACadSharp (опционально, лучшее качество для `.dwg`)

ACadSharp — .NET-библиотека, читающая и пишущая `.dwg` напрямую, без
промежуточной конвертации. Если она собрана, приложение использует её
в первую очередь.

Сборка (нужен **.NET SDK 6.0+**):
```cmd
cd prodact\acadsharp\AcadReplace
dotnet publish -c Release -r win-x64 --self-contained
```
Подробности и точные пути — в `prodact/acadsharp/BUILD.md`. Собранную утилиту
модуль `acadsharp_native.py` находит автоматически.

> Если ACadSharp не собирать — ничего страшного: `.dwg` будет обрабатываться
> через LibreDWG.

## 6. Генерация иконок

Цветные PNG-иконки и `app.ico` уже лежат в `resources/`. Пересоздать их
(например, после правки `tools/make_icons.py`) можно так:

```cmd
python tools\make_icons.py
```

Для иконки самого EXE используется `resources\app.ico` (она прописана в `.spec`).
Если меняли `app.png`, пересоздайте `.ico`:

```cmd
python -c "from PIL import Image; im=Image.open('resources/icons/app.png').convert('RGBA'); im.save('resources/app.ico', sizes=[(16,16),(32,32),(48,48),(64,64),(128,128),(256,256)])"
```

## 7. Сборка через PyInstaller

Сборка управляется файлом `text_replacer.spec`, в котором уже настроено:

- включение папок `resources/` и `libredwg/` в EXE (`datas`);
- сбор нативных частей PDF-библиотек и темы `sv_ttk` через `collect_all`;
- `hiddenimports` для модулей, которые PyInstaller не видит автоматически;
- `console=False` — без чёрного консольного окна;
- иконка `resources/app.ico`.

Запуск из корня проекта:

```cmd
python -m PyInstaller --clean --noconfirm text_replacer.spec
```

Результат: **`dist\ЗаменительТекста.exe`** — один файл.

> **Важно про `.dwg`:** даже в «однофайловом» режиме утилиты LibreDWG
> распаковываются во временную папку при запуске. Если вы раздаёте программу,
> надёжнее держать папку `libredwg\` **рядом с .exe** — приложение ищет её и там.

## 8. Проверка результата

1. Запустите `dist\ЗаменительТекста.exe` — должно открыться окно с темой Sun Valley.
2. Проверьте меню **Помощь → Справка** и **О программе** (должен быть указан автор).
3. Сделайте тестовую замену:
   - создайте папку с `.docx`, `.xlsx`, `.pdf`, `.dwg`;
   - добавьте задачу, отметьте цели, нажмите «Выполнить замену»;
   - проверьте журнал и работу кнопки «Вернуть прежнее».

## 9. Частые ошибки и их решение

| Симптом | Причина | Решение |
|---------|---------|---------|
| `ModuleNotFoundError: sv_ttk` при запуске EXE | тема не вошла в сборку | убедитесь, что `sv-ttk` установлен и есть в `hiddenimports`/`collect_all` (в `.spec` уже есть) |
| Окно открывается «голым», без темы | `sv_ttk` не найден в рантайме | переустановите `sv-ttk`, пересоберите с `--clean` |
| `.dwg` не обрабатывается, ошибка «утилиты LibreDWG не найдены» | нет папки `libredwg\` | положите `libredwg\` рядом с `.exe` (см. раздел 5.1) |
| Иконки не отображаются | папка `resources` не вошла | проверьте `datas` в `.spec`; пересоберите |
| PDF не меняется ни одной библиотекой | нативные DLL PyMuPDF не вошли | в `.spec` используется `collect_all("fitz"/"pymupdf")` — переустановите `pymupdf` |
| Антивирус удаляет EXE | ложное срабатывание на PyInstaller | подпишите EXE или добавьте в исключения; это типично для упакованных Python-приложений |
| EXE очень большой | вшиты все библиотеки | это нормально (PDF-движки тяжёлые); можно включить `upx` (уже включён) |
| `dotnet` не найден при сборке ACadSharp | нет .NET SDK | установите .NET SDK 6.0+ или пропустите ACadSharp (будет LibreDWG) |
| Кириллица в путях ломает LibreDWG | особенности утилит | по возможности избегайте кириллицы в путях к временным файлам |

## 10. Структура проекта

```
text_replacer_app/
├── main.py                  # точка входа, меню, шапка
├── text_replacer.py         # вся логика замены и отката
├── app_utils.py             # каталоги приложения
├── acadsharp_native.py      # обёртка ACadSharp (если собрана)
├── dxfrw_native.py          # обёртка libdxfrw (если собрана)
├── requirements.txt
├── text_replacer.spec       # конфигурация сборки EXE
├── ui/
│   ├── theme.py             # тема Sun Valley + запасная
│   ├── text_replace_tab.py  # основной экран (список задач)
│   ├── task_dialog.py       # всплывающее окно задачи
│   ├── about.py             # справка и «О программе»
│   └── icons.py             # загрузка PNG-иконок
├── tools/
│   └── make_icons.py        # генератор цветных иконок
├── resources/
│   ├── app.ico              # иконка EXE
│   └── icons/*.png          # цветные иконки интерфейса
├── libredwg/                # ВНЕШНЕЕ: утилиты + DLL для .dwg
└── prodact/                 # ВНЕШНЕЕ: исходники LibreDWG/ACadSharp
```

В репозитории **Github** папки `libredwg/` (бинарники) и `prodact/`
(тяжёлые исходники сторонних библиотек) исключены — см. `Github/README.md`,
где описано, откуда их взять для полноценной сборки `.dwg`.
